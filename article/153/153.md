# 迁移浏览器数据的方法

本文是在同一个浏览器的迁移。浏览器为了抢用户，都支持导入其他家的数据。  
迁移浏览器数据，是指把浏览器的书签、历史记录、密码、设置、扩展的程序和数据、cookie、缓存、网站设置（权限）、已打开的标签页和窗口等数据，迁移到另一个位置（新浏览器实例、新电脑）的操作。  
本文介绍 Windows 系统下 Chrome、Firefox 两个浏览器的数据迁移操作。  

## Chrome

**测试版本：145**  

### 官方同步

登录 Google 账号后，可以同步这些数据（ [来源](https://support.google.com/chrome/answer/165139?hl=zh-Hans) ）：  

- 书签、历史记录、阅读请单、标签页、标签页分组
- 扩展的程序
- 密码
- 设置、主题

**不能同步的数据**：

- cookie
- 扩展的数据
- 网站设置（权限）
- 缓存

缺的数据还是挺多的，算不上「迁移」。  

### 迁移本地文件

Chrome 的配置与缓存文件都在这个路径：

```
%LocalAppData%\Google\Chrome\User Data
```

如果没有多用户的话，默认用的就是这个路径下的 `Default` 文件夹；如果创建了多个「用户 / 个人资料」，就还会有 `Profile 1`、`Profile 2` 等文件夹。  
复制粘贴的图形操作就不说了，这里给出命令行复制的命令：  

```
# PowerShell 版本
robocopy "$env:LocalAppData\Google\Chrome\User Data" "<目标路径>\User Data" /E /COPYALL /XJ /MT:16 /R:1 /W:1 /NP /LOG:"%TEMP%\chrome_copy.log"

# CMD 版本
robocopy "%LocalAppData%\Google\Chrome\User Data" "<目标路径>\User Data" /E /COPYALL /XJ /MT:16 /R:1 /W:1 /NP /LOG:"%TEMP%\chrome_copy.log"
```

复制这个文件夹之后，书签、历史记录、密码、设置、缓存、扩展的数据、标签页等等，就直接迁移了。  
但部分数据需要单独处理：  

- cookie
- 扩展程序（在 144 及之前的版本，这一项还不会被重置）
- 启动页（设置 → 起始页面）、主页、固定标签页
- 默认搜索引擎

![](./Chrome%20重置了这些设置.png)  

[Chrome 重置了这些设置 ](https://support.google.com/chrome?p=ui_automatic_settings_reset)

这些数据是与操作系统用户的密钥相关的，由于安全原因，复制后不能直接使用。  

**扩展程序**  
由于被重置，**扩展需要重新安装，扩展在浏览器里的设置也要重新设**（比如 Tampermonkey 要重新设置「允许运行用户脚本」）。但好在扩展的数据是可以直接恢复的，重装扩展之后，先禁用再启用，扩展就会读取 `Local Extension Settings` 文件夹里的数据。  

**cookie**  
直接复制后也不能用。  
有一个开源项目 [CookieCloud](https://github.com/easychen/CookieCloud) 可以把 cookie 加密同步到服务器，再同步到另一个浏览器。服务器需要自建，或者使用网友免费提供的。我没有测试这个软件，这里仅做介绍，**不担保其安全性**。  

## Firefox

### 官方同步

登录 Mozilla 账号后，可以同步这些数据（ [来源](https://support.mozilla.org/zh-CN/kb/how-do-i-manage-my-firefox-sync-account) ）：  

- 书签、历史记录、标签页
- 扩展的程序（附加组件）
- 密码
- 设置、主题

**不能同步的数据**：

- cookie
- 扩展的数据
- 网站设置（权限）
- 缓存

由于 Firefox 复制文件之后是全部数据都能迁移的，所以没必要用官方同步。  

### 迁移本地文件

**找到配置文件**  
Firefox 的配置与缓存是分开存储的。  
配置文件 在 `Roaming`  
```
%AppData%\Mozilla\Firefox\Profiles\
```
缓存 在 `Local`  
```
%LocalAppData%\Mozilla\Firefox\Profiles\
```

【】演示  
Profiles 里面，每一个文件夹就对应一套配置。打开之后可能发现里面不止一个文件夹。默认的配置文件名是：`<小写字母数字>.default-release` 或 `<小写字母数字>.default `。那怎么知道要迁移哪个呢？  
打开一个新标签页，在地址栏输入 `about:support` （菜单 → 帮助 → 更多故障排除信息）。在第一部分「应用程序概要」有一项「配置文件文件夹」。这里显示的就是当前浏览器使用的配置文件，需要注意最后一个文件夹的名字。

**关闭 Firefox**  

**复制文件**  

**创建配置文件、设定路径**
Windows + R 键，运行 `firefox.exe -P` 打开「**配置文件管理器**」。（用这个方式创建的配置文件，任务栏的标题是：`<网页标题> —— Mozilla Firefox` 。）

- 可以起一个自己喜欢的名字，也可以继续用 default。
- 可以自定义配置文件的路径。注意这里是直接使用选定的文件夹，Firefox 不会再建一个。

创建好新的配置文件夹之后，**先不要启动 Firefox**，退出「配置文件管理器」。  

**迁移配置文件**  
把旧的配置、缓存复制过去就完成迁移了。  
**粘贴时要注意：不要**直接把整个旧的 配置文件夹（<小写字母数字>.default）复制到 Firefox\Profiles\ 这个路径。而应该把旧的 配置文件夹（<小写字母数字>.default）的子文件夹及文件，复制到新的 profile 里面——**只复制文件夹的内容，不复制文件夹本身**。因为用户配置文件夹的名字（包括随机的 8 位数字字符串）必须完全匹配才有效。

**启动 Firefox**  
Windows + R 键，运行 `firefox.exe -P` 打开「**配置文件管理器**」。  
「启动时不询问并使用选定的配置文件」：如果不勾选这一项，每次启动 Firefox 都会弹出这个，要求选择配置文件。  

如果以上内容没讲清楚，请看 Firefox 官方文档：  
[备份你的信息](https://support.mozilla.org/zh-CN/kb/%E5%A4%87%E4%BB%BD%E4%BD%A0%E7%9A%84%E4%BF%A1%E6%81%AF)  
[从旧的配置文档中恢复重要数据](https://support.mozilla.org/zh-CN/kb/%E4%BB%8E%E6%97%A7%E7%9A%84%E9%85%8D%E7%BD%AE%E6%96%87%E6%A1%A3%E4%B8%AD%E6%81%A2%E5%A4%8D%E9%87%8D%E8%A6%81%E6%95%B0%E6%8D%AE)

## 多用户浏览器的情况

多用户浏览器是指让浏览器以另一个用户的身份（低权限）运行。  

**关于文件权限**  
如果你的多用户方案搭配了 ACL，那你可能会担心本地文件复制后的权限问题。  
我的操作：由于「上网用户」无法把文件复制到 `C:\Users\Public` 这个公用文件夹（可能是因为属于 Users 组）。我是直接给「上网用户」添加了目标文件夹的访问权限。复制之后「主用户」直接就可以打开，不需要再设置权限。  
建议的操作：压缩。这里给出用 7z 压缩的命令：
```
# PowerShell 版本
& "$env:ProgramFiles\7-Zip\7z.exe" a -tzip -mx1 -mmt=on "$zipDestination" "$sourceDir\*"
# （命令开头的 & 不能省）
```
